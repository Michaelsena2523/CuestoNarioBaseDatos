Conceptos básicos

1.¿Qué es un Dato?

Un dato es una representación simbólica (numérica, alfabética, algorítmica, espacial, etc.) de un atributo o variable cuantitativa o cualitativa. En términos simples, un dato es cualquier conjunto de caracteres que tiene algún significado para quien lo observa o lo utiliza. Los datos pueden ser de diferentes tipos, como números, textos, imágenes, sonidos, etc. Cuando los datos se procesan, organizan y se les da un contexto, se convierten en información, la cual puede ser utilizada para tomar decisiones, realizar análisis o comprender fenómenos.

2.¿Qué es información?

La información es el conjunto de datos procesados y organizados de tal manera que tienen un significado o valor para quien los recibe. Es el resultado de procesar, interpretar y estructurar datos de manera que sean útiles para la toma de decisiones, la resolución de problemas, el análisis, y la comunicación.

3.¿Diferencia entre dato e información?

La diferencia entre dato e información se basa en el procesamiento y el contexto que se les da. Aquí están las diferencias clave:

Dato:

Es un valor bruto, sin procesar, que por sí solo puede no tener significado. Por ejemplo, un número aislado como "25".

Información:

Es el resultado de procesar y contextualizar los datos, dándoles un significado útil. Por ejemplo, "La temperatura promedio en la ciudad es de 25°C". Contexto:

Dato:

Carece de contexto. Es simplemente una representación simbólica sin un propósito claro por sí mismo.

Información:

Tiene contexto y relevancia. Es significativa y puede ser utilizada para la toma de decisiones o para entender una situación. Utilidad:

Dato:

Puede no ser útil por sí solo. Necesita ser interpretado o procesado para tener un valor real.

Información:

Es útil y aplicable para un propósito específico. Ayuda en la comprensión, análisis y toma de decisiones.

5.¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear una representación visual o abstracta de un sistema de información para organizar y estructurar los datos de manera que sean fácilmente comprensibles y utilizables. Este proceso implica definir y analizar los requisitos de datos necesarios para apoyar los procesos de negocio de una organización.

6.¿Tipos de bases de Datos, según:
  • - Variabilidad
  • - Contenido
  • - Modelo de base de datos

    • Bases de Datos según Variabilidad:


    Estáticas:
  • Almacenan datos que cambian poco o casi nunca.

  • Útiles para datos históricos o de referencia que no requieren actualizaciones frecuentes.

  • Ejemplo: Archivos de registros históricos.


    • Dinámicas:
  • Contienen datos que se actualizan con regularidad o en tiempo real.

  • Esenciales para aplicaciones y sistemas que necesitan datos actualizados continuamente.

  • Ejemplo: Sistemas de gestión de inventarios en tiempo real.

    • Bibliográficas:


  • Almacenan referencias y metadatos de libros, artículos y documentos.

  • Ejemplo: Catálogos académicos y bibliotecas digitales.

    • Multimediales:
  • Gestionan imágenes, videos y audio.

  • Ejemplo: Galerías en línea y bibliotecas de medios.

    • Geoespaciales:
  • Almacenan datos geográficos y espaciales.

  • Ejemplo: Sistemas de navegación y mapas digitales.

    • Numéricas:
  • Almacenan datos numéricos para cálculos y análisis.

  • Ejemplo: Aplicaciones científicas y financieras.

    • Modelos de Base de Datos:


    Relacional:
  • Organiza los datos en tablas con filas y columnas.

  • Utiliza claves primarias y foráneas para establecer relaciones.

  • Ejemplos: MySQL, PostgreSQL, Oracle.

    • NoSQL:


  • Maneja datos no estructurados o semi-estructurados.

  • Empleado en aplicaciones que requieren escalabilidad y flexibilidad.

  • Ejemplos: MongoDB, Cassandra, Neo4j.

    • Orientado a Objetos:
  • Representa datos como objetos similares a la programación orientada a objetos.

  • Utilizado en aplicaciones con estructuras de datos complejas.

  • Ejemplos: ObjectDB, db4o.

    • Tiempo Real:
  • Diseñado para datos que necesitan procesamiento y respuesta inmediata.
  • Utilizado en sistemas de monitoreo en tiempo real y transacciones financieras rápidas.

    • 7.¿Qué es el modelo relacional?

    El modelo relacional es un enfoque para gestionar bases de datos donde los datos se organizan en tablas (o relaciones). Cada tabla está compuesta por filas (tuplas) y columnas (atributos), y las relaciones entre las tablas se establecen mediante claves primarias y claves foráneas. Este modelo permite operaciones como insertar, actualizar, eliminar y consultar datos utilizando el lenguaje SQL (Structured Query Language). Las bases de datos relacionales son conocidas por su flexibilidad, integridad de datos y capacidad para manejar grandes volúmenes de información de manera eficiente. Ejemplos incluyen MySQL, PostgreSQL y Oracle.

    9.¿Qué es una entidad?

    Una entidad, en el contexto de las bases de datos y el modelado de datos, representa un objeto o concepto del mundo real que se desea gestionar y sobre el cual se desea almacenar información dentro de un sistema informático. Las entidades tienen características o atributos que describen aspectos específicos de ese objeto. Por ejemplo, en una base de datos de una biblioteca, las entidades podrían incluir libros, autores, usuarios y préstamos, cada una con sus propios atributos relevantes (como título del libro, nombre del autor, número de usuario, fecha de préstamo, etc.).

    10.¿Qué es un atributo y sus tipos?

    Un atributo, en el contexto de las bases de datos y el modelado de datos, es una característica o propiedad específica que describe una entidad. Los atributos son las unidades básicas de datos que representan detalles sobre las entidades en una base de datos.

    Tipos de Atributos:

    Atributos Simples:
  • Representan datos indivisibles y elementales.
  • Ejemplos: Nombre, Edad, Precio.

    • Atributos Compuestos:
  • Son combinaciones de atributos simples.
  • Ejemplo: Dirección (compuesta por calle, ciudad, país).

    • Atributos Monovalentes:
  • Tienen un solo valor para una entidad específica.
  • Ejemplo: Fecha de Nacimiento.

    • Atributos Multivaluados:
  • Pueden tener múltiples valores para una entidad.
  • Ejemplo: Números de teléfono de un cliente.

    • Atributos Derivados:
  • Se calculan o derivan de otros atributos.
  • Ejemplo: Edad, calculada a partir de la fecha de nacimiento.

    • Atributos Clave:
  • Identifican de manera única a una entidad dentro de una relación.
  • Ejemplo: Número de Identificación de un cliente.

    • 11.¿Qué es una relación?

    Una relación en bases de datos describe cómo dos o más entidades están conectadas entre sí. Define cómo las entidades se relacionan o interactúan dentro del sistema de base de datos, especificando la naturaleza y el tipo de conexión que existe entre ellas.

    12.¿Qué son las llaves primarias, foránea y sus características?

    Las llaves primarias y foráneas son conceptos clave en el diseño de bases de datos relacionales para establecer y mantener la integridad de los datos. Aquí tienes una descripción breve de cada uno y sus características:

    Llave Primaria (Primary Key):


    Definición:Una llave primaria es un atributo o conjunto de atributos que identifican de manera única a cada fila en una tabla.
    Características:
  • Debe contener valores únicos para cada registro en la tabla.
  • No puede contener valores nulos (NULL).
  • Puede estar compuesta por uno o más atributos.
  • Proporciona una forma rápida de acceder a los datos mediante búsquedas por su valor único.

    • Llave Foránea (Foreign Key):

    Definición:Una llave foránea es un campo o conjunto de campos en una tabla que hace referencia a la llave primaria de otra tabla.
    Características:
  • Establece una relación entre dos tablas, permitiendo la integridad referencial.
  • El valor de la llave foránea en una tabla debe coincidir con un valor existente en la llave primaria correspondiente de otra tabla.
  • Ayuda a mantener la consistencia de los datos y evita acciones que podrían afectar la integridad de la base de datos, como eliminar accidentalmente datos relacionados.
  • Puede ser nulo si se permite en la estructura de la tabla, indicando que no hay relación establecida en ese campo.

    • 13.¿Qué son las reglas del Negocio?

    Las reglas del negocio en bases de datos son normas y restricciones que dictan cómo deben manejarse y utilizar los datos dentro de una organización. Estas reglas aseguran la integridad de los datos, definen validaciones necesarias, cumplen con normativas legales y guían operaciones específicas relacionadas con los datos y procesos de la organización.

    14.¿Qué es la cardinalidad o mapeo?

    La cardinalidad en bases de datos describe cuántas veces una entidad puede relacionarse con otra entidad:
    Uno a Uno (1:1): Cada instancia de una entidad se relaciona exactamente con una instancia de la otra entidad, y viceversa.
    Uno a Muchos (1): Cada instancia de una entidad puede relacionarse con varias instancias de la otra entidad, pero cada instancia de esta última se relaciona solo con una instancia de la primera entidad.
    Muchos a Uno (N:1): Varias instancias de una entidad se relacionan con una sola instancia de la otra entidad.
    Muchos a Muchos (N): Varias instancias de una entidad pueden relacionarse con varias instancias de la otra entidad, necesitando una tabla de unión para gestionar esta relación en una base de datos relacional.
